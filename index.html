📦 第一部分：准备工作与系统要求

1.1 环境说明

本指南适用于以下环境：

  • 主机系统：Ubuntu 24.04 LTS（你的当前系统）
  • 虚拟机软件：VirtualBox（开源免费）
  • 虚拟机系统：Ubuntu 24.04 LTS（与主机相同版本，兼容性好）
  • 目标应用：OpenClaw AI 助手

为什么选择虚拟机隔离？

  • ✅ OpenClaw 运行在独立环境中，不影响主机系统
  • ✅ 可以随时创建快照，出问题可快速恢复
  • ✅ 网络隔离，保护主机数据安全
  • ✅ 方便测试和实验

1.2 系统要求检查

首先检查你的主机是否满足要求：

# 检查 CPU 信息 lscpu | grep -E "^CPU\(s\)|^Model name" # 检查内存 free -h # 检查磁盘空间 df -h /
项目 最低要求 推荐配置
主机内存 (RAM) 8 GB 16 GB 或更多
可用磁盘空间 30 GB 60 GB 或更多
CPU 核心 4 核 6 核或更多
虚拟机内存分配 4 GB 6-8 GB

重要：确保主机至少有 8GB 内存！如果你只有 8GB，建议给虚拟机分配 4GB，留 4GB 给主机。如果内存不足，系统会变得非常卡顿。

1.3 下载所需文件

需要下载 Ubuntu 24.04 ISO 镜像文件：

# 方法1：使用 wget 下载（推荐，国内镜像源） cd ~/Downloads wget https://mirrors.tuna.tsinghua.edu.cn/ubuntu-releases/24.04/ubuntu-24.04.3-desktop-amd64.iso # 方法2：使用 curl 下载 curl -L -o ubuntu-24.04.3-desktop-amd64.iso https://mirrors.tuna.tsinghua.edu.cn/ubuntu-releases/24.04/ubuntu-24.04.3-desktop-amd64.iso

下载地址备选：

  • 清华镜像：https://mirrors.tuna.tsinghua.edu.cn/ubuntu-releases/24.04/
  • 阿里镜像：https://mirrors.aliyun.com/ubuntu-releases/24.04/
  • 官方地址：https://ubuntu.com/download/desktop

ISO 文件大小约 4.5GB，下载可能需要一些时间。

🖥️ 第二部分：安装 VirtualBox 虚拟机软件

2.1 安装 VirtualBox

在 Ubuntu 24.04 主机上安装 VirtualBox：

1

更新软件包列表

sudo apt update
2

安装 VirtualBox 和扩展包

# 安装 VirtualBox sudo apt install -y virtualbox virtualbox-ext-pack # 如果上面的命令报错，使用以下方法安装最新版本： wget -q https://www.virtualbox.org/download/oracle_vbox_2016.asc -O- | sudo apt-key add - echo "deb [arch=amd64] https://download.virtualbox.org/virtualbox/debian $(lsb_release -sc) contrib" | sudo tee /etc/apt/sources.list.d/virtualbox.list sudo apt update sudo apt install -y virtualbox-7.0
3

将当前用户添加到 vboxusers 组（重要！）

sudo usermod -aG vboxusers $USER

执行后需要注销并重新登录才能生效。

4

验证安装

# 检查 VirtualBox 版本 vboxmanage --version # 启动 VirtualBox 图形界面（可选） virtualbox &

2.2 安装 VirtualBox Extension Pack（推荐）

扩展包提供 USB 2.0/3.0 支持、RDP、磁盘加密等功能：

# 查看已安装的 VirtualBox 版本 VBoxManage --version | cut -d r -f 1 # 下载对应版本的扩展包（假设是 7.0.x） wget https://download.virtualbox.org/virtualbox/7.0.18/Oracle_VM_VirtualBox_Extension_Pack-7.0.18.vbox-extpack # 安装扩展包 sudo VBoxManage extpack install Oracle_VM_VirtualBox_Extension_Pack-*.vbox-extpack

扩展包版本必须与 VirtualBox 版本完全匹配！请根据 vboxmanage --version 的输出下载对应版本。

🐧 第三部分：创建虚拟机并安装 Ubuntu

3.1 创建虚拟机

你可以使用图形界面或命令行创建虚拟机。这里提供两种方法：

方法 A：图形界面创建（推荐新手）

1

启动 VirtualBox

virtualbox &
2

点击「新建」按钮

填写以下信息：

  • 名称：OpenClaw-VM
  • 类型：Linux
  • 版本：Ubuntu (64-bit)
3

分配内存

建议：4096 MB（4GB）或更多（取决于你的主机内存）

4

创建虚拟硬盘

  • 选择「现在创建虚拟硬盘」
  • 硬盘文件类型：VDI (VirtualBox 磁盘映像)
  • 存储方式：动态分配（推荐，节省空间）
  • 大小：50 GB 或更多

方法 B：命令行创建（快速）

# 创建虚拟机 VBoxManage createvm --name "OpenClaw-VM" --register # 设置内存和 CPU，请根据需要自行修改内存和CPU数量（默认4G+2核） VBoxManage modifyvm "OpenClaw-VM" --memory 4096 --cpus 2 # 创建虚拟硬盘，请根据需要自行修改硬盘大小（默认51200） VBoxManage createhd --filename ~/VirtualBox\ VMs/OpenClaw-VM/disk.vdi --size 51200 # 添加 SATA 控制器和硬盘 VBoxManage storagectl "OpenClaw-VM" --name "SATA Controller" --add sata VBoxManage storageattach "OpenClaw-VM" --storagectl "SATA Controller" --port 0 --device 0 --type hdd --medium ~/VirtualBox\ VMs/OpenClaw-VM/disk.vdi # 挂载 ISO 镜像 VBoxManage storagectl "OpenClaw-VM" --name "IDE Controller" --add ide VBoxManage storageattach "OpenClaw-VM" --storagectl "IDE Controller" --port 0 --device 0 --type dvddrive --medium ~/Downloads/ubuntu-24.04.3-desktop-amd64.iso

建议通过VirtualBox可视化界面：设置 —— 显示，将显存调到128MB，防止虚拟机系统卡顿。

3.2 配置虚拟机网络（重要！）

为了实现网络隔离，我们使用 NAT 模式：

# 设置网络为 NAT 模式（默认，虚拟机可通过主机上网，但外部无法直接访问虚拟机） VBoxManage modifyvm "OpenClaw-VM" --nic1 nat # 如果需要从主机 SSH 连接虚拟机，需要设置端口转发 VBoxManage modifyvm "OpenClaw-VM" --natpf1 "ssh,tcp,,2222,,22"

网络模式说明：

  • NAT（推荐）：虚拟机通过主机上网，外部无法直接访问虚拟机，实现网络隔离
  • 桥接：虚拟机获得独立 IP，与主机在同一网络，隔离性较差
  • Host-Only：虚拟机只能与主机通信，无法上网
  • Internal：完全隔离，虚拟机之间可通信，但无法上网

3.3 启动虚拟机并安装 Ubuntu

# 启动虚拟机（图形界面模式） VBoxManage startvm "OpenClaw-VM" # 或使用无头模式（后台运行，需要远程连接） VBoxManage startvm "OpenClaw-VM" --type headless
1

选择安装语言

建议选择「中文(简体)」或「English」

2

选择「安装 Ubuntu」

3

键盘布局

选择「Chinese」或「US」

4

安装类型

选择「正常安装」

5

安装方式

选择「清除整个磁盘并安装 Ubuntu」（这是虚拟磁盘，不会影响主机）

6

设置用户信息（请牢记！）

  • 您的姓名：openclaw
  • 计算机名：openclaw-vm
  • 用户名：openclaw
  • 密码：设置一个你记得住的密码

请牢记用户名和密码！后续操作会频繁使用。

7

等待安装完成

安装过程约 15-30 分钟。安装完成后，重启虚拟机。

⚙️ 第四部分：虚拟机系统基础配置

4.1 更换国内软件源（加速下载）

# 备份原始源文件 sudo cp /etc/apt/sources.list /etc/apt/sources.list.backup # 调整至root用户 su root # 如未初始化root用户密码，请执行初始化操作，再进入root模式 sudo passwd su root # 使用阿里云源 sudo cat > /etc/apt/sources.list << 'EOF' deb http://mirrors.aliyun.com/ubuntu/ focal main restricted universe multiverse deb http://mirrors.aliyun.com/ubuntu/ focal-security main restricted universe multiverse deb http://mirrors.aliyun.com/ubuntu/ focal-updates main restricted universe multiverse deb http://mirrors.aliyun.com/ubuntu/ focal-backports main restricted universe multiverse EOF # 返回到原始用户 su \<用户名\> # 更新软件列表 sudo apt update && sudo apt upgrade -y

4.2 安装基础工具

sudo apt install -y curl wget git vim net-tools openssh-server build-essential

4.3 安装 Node.js（OpenClaw 依赖）

OpenClaw 需要 Node.js 22.14+ 或 Node.js 24。推荐使用 nvm 安装：

# 安装 nvm curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash # 使 nvm 生效 source ~/.bashrc # 如果 source 无效，重新打开终端或执行： export NVM_DIR="$HOME/.nvm" [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" # 安装 Node.js 24 nvm install 24 nvm use 24 nvm alias default 24 # 验证安装 node -v npm -v

如果 nvm 安装失败（网络问题），使用以下备用方法：

# 使用 NodeSource 仓库安装 curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt install -y nodejs # 验证 node -v npm -v

4.4 配置 npm 国内镜像（可选）

# 使用淘宝镜像加速 npm 下载 npm config set registry https://registry.npmmirror.com # 验证 npm config get registry

🔒 第六部分：虚拟机隔离与安全设置

这部分非常重要！确保虚拟机中的操作不会影响你的主机。

6.1 网络隔离设置

1

确认使用 NAT 模式

NAT 模式下，虚拟机通过主机的网络连接上网，但外部网络无法直接访问虚拟机。

# 检查网络模式 VBoxManage showvminfo "OpenClaw-VM" | grep "NIC 1" # 如果不是 NAT，修改为 NAT VBoxManage modifyvm "OpenClaw-VM" --nic1 nat
2

禁用共享文件夹（如果不需要）

# 移除所有共享文件夹 VBoxManage sharedfolder remove "OpenClaw-VM" --name # 或在图形界面：设置 -> 共享文件夹 -> 无
3

禁用拖放和剪贴板（可选，更高安全性）

# 禁用拖放和剪贴板 VBoxManage modifyvm "OpenClaw-VM" --draganddrop disabled VBoxManage modifyvm "OpenClaw-VM" --clipboard disabled

6.2 虚拟机内部防火墙

在虚拟机内部配置防火墙，进一步增强隔离：

# 在虚拟机内执行： # 安装 UFW 防火墙 sudo apt install -y ufw # 启用防火墙 sudo ufw enable # 只允许 SSH（如果需要） sudo ufw allow ssh # 查看状态 sudo ufw status verbose

6.3 创建快照（重要保护）

快照可以保存虚拟机的当前状态，如果出现问题可以快速恢复。

# 创建快照 VBoxManage snapshot "OpenClaw-VM" take "OpenClaw-Installed" --description "OpenClaw 安装完成后的干净状态" # 查看快照列表 VBoxManage snapshot "OpenClaw-VM" list # 恢复到快照（如果需要） VBoxManage snapshot "OpenClaw-VM" restore "OpenClaw-Installed" # 删除快照 VBoxManage snapshot "OpenClaw-VM" delete "OpenClaw-Installed"

建议：在进行任何可能影响系统的操作之前，先创建一个快照！这样如果出现问题，可以立即恢复到正常状态。

6.4 完全隔离模式（最高安全）

如果你需要最大程度的隔离，可以使用 Internal 网络模式：

# 设置为内部网络模式（虚拟机无法上网） VBoxManage modifyvm "OpenClaw-VM" --nic1 intnet

注意：Internal 模式下，虚拟机无法访问外部网络，OpenClaw 将无法连接 API。除非有特殊安全需求，否则不建议使用此模式。

6.5 隔离设置总结

隔离级别 设置 影响
基础（推荐） NAT 网络模式 虚拟机可上网，外部无法直接访问虚拟机
中等 禁用共享文件夹、剪贴板 无法在主机和虚拟机间直接传输文件
高级 启用虚拟机防火墙 限制虚拟机内部网络访问
最高 Internal 网络模式 虚拟机完全无法上网（不适用于 OpenClaw）

❓ 第七部分：常见问题与故障排除

7.1 VirtualBox 相关问题

问题：VirtualBox 启动报错 "Kernel driver not installed"

# 重新加载内核模块 sudo /sbin/vboxconfig # 如果仍然失败，重新安装 sudo apt remove --purge virtualbox* sudo apt install -y virtualbox virtualbox-ext-pack sudo usermod -aG vboxusers $USER # 注销并重新登录

问题：虚拟机无法启动，提示 VT-x/AMD-V 错误

# 检查 CPU 是否支持虚拟化 egrep -c '(vmx|svm)' /proc/cpuinfo # 如果返回 0，说明 CPU 不支持虚拟化或未在 BIOS 中启用 # 启用嵌套虚拟化（如果在虚拟机中运行） VBoxManage modifyvm "OpenClaw-VM" --nested-hw-virt on

问题：虚拟机运行缓慢

  • 增加虚拟机内存（如果主机有足够内存）
  • 增加虚拟机 CPU 核心数
  • 启用 3D 加速：设置 -> 显示 -> 启用 3D 加速
  • 为虚拟机磁盘启用 SSD 模拟：设置 -> 存储 -> 勾选「固态驱动器」

7.2 OpenClaw 相关问题

问题：openclaw 命令找不到

# 检查 Node.js 是否安装 node -v # 检查 npm 全局 bin 目录 npm prefix -g # 检查 PATH echo "$PATH" # 如果 npm bin 目录不在 PATH 中，添加： echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.bashrc source ~/.bashrc

问题：npm 安装缓慢或失败

# 使用国内镜像 npm config set registry https://registry.npmmirror.com # 验证 npm config get registry

问题：OpenClaw 启动失败

# 查看详细错误日志 openclaw logs --tail 100 # 检查配置 openclaw doctor # 重新运行配置向导 openclaw onboard # 完全重装 npm uninstall -g openclaw npm install -g openclaw@latest openclaw onboard --install-daemon

7.3 网络相关问题

问题：虚拟机无法上网

# 检查网络接口 ip addr show # 检查 DNS ping 8.8.8.8 ping google.com # 如果 DNS 问题，编辑 resolv.conf sudo vim /etc/resolv.conf # 添加： # nameserver 8.8.8.8 # nameserver 114.114.114.114

问题：从主机 SSH 连接虚拟机失败

# 确保设置了端口转发 VBoxManage modifyvm "OpenClaw-VM" --natpf1 "ssh,tcp,,2222,,22" # 确保虚拟机内 SSH 服务运行 sudo systemctl status ssh # 从主机连接 ssh -p 2222 openclaw@localhost
